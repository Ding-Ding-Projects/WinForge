C# and XAML are behavioral-oracle source only during migration; the native executable does not host the CLR, wrap the managed app, use C++/CLI, or outsource behavior over IPC. · WinForge 正喺遷移去真正嘅 C++20/C++/WinRT WinUI 3 app。遷移期間 C# 同 XAML 只係行為基準來源；原生 executable 唔會 host CLR、包住受控 app、用 C++/CLI，或者用 IPC 外判行為。

## Native regex search and builder · 原生正規搜尋同建立器

`RegexSearchSurface.h` registers Shell catalog, All Apps, cached Package Discover, and the static local Regex Cheatsheet as the complete current native regex-search set. Each defaults to literal filtering and uses bounded non-JIT PCRE2-16 only through an explicit Regex mode. The full four-step `module.regextester` builder handles flags, escaped literal/route/package/version recipes, safe composition, groups/alternation/word-boundary/lookaround assertions/quantifiers, capture preview, and valid-only application to a chosen native target. Package regex is cache-only, never enters argv/HTTPS, disables remote Search, forces Discover when applied through the builder, and its `NativePackageQueryAudit` changes only for a real package query. PCRE2 attribution ships in `THIRD-PARTY-NOTICES.txt`, the portable ZIP, and the installer. · `RegexSearchSurface.h` 登記咗 Shell 目錄、所有 app、已快取 Package Discover 同靜態本機 Regex Cheatsheet 做而家完整原生 regex-search 集合；每個預設 literal 篩選，只有經明確 Regex mode 先會用有界、非 JIT PCRE2-16。完整四步 `module.regextester` 建立器處理旗標、已 escape literal／路線／套件／版本 recipe、安全組合、group／alternation／word-boundary／lookaround assertion／quantifier、capture 預覽，同只會套用有效模式去已揀原生目標。Package regex 只限快取、唔會入 argv／HTTPS、會停用遠端 Search、由 builder 套用時會強制 Discover，而且 `NativePackageQueryAudit` 只會喺真正套件查詢時改。PCRE2 歸屬會放喺 `THIRD-PARTY-NOTICES.txt`、可攜 ZIP 同 installer。

## Native UUID v7 · 原生 UUID v7

`module.uuidv7` is a native C++ RFC 9562 generator/decoder, available through `uuidv7` and its canonical route. It bulk-generates 1–1000 canonical values from the 48-bit Unix-millisecond field and cryptographic randomness; optional monotonic mode handles same-millisecond creation, backwards clocks, and `rand_a` overflow. The local decoder accepts canonical, compact, braced, and `urn:uuid:` text; reports version/variant and v7/best-effort-v1 UTC/local time; canonicalizes lowercase output; and clears stale results after invalid input. Explicit Copy is the only clipboard effect, and state survives language changes. The route is `in-progress` only because its fresh visual capture is blocked. · `module.uuidv7` 係原生 C++ RFC 9562 產生／解碼器，可以用 `uuidv7` 同本體 route 開啟。佢由 48-bit Unix millisecond 欄位加密碼學隨機值批量產生 1–1000 條標準值；可選 monotonic 模式處理同一毫秒建立、時鐘倒退同 `rand_a` overflow。本機 decoder 接受標準、緊湊、花括號同 `urn:uuid:` 文字；報告 version／variant 同 v7／best-effort-v1 UTC／本機時間、標準化小楷輸出，並喺無效輸入後清除舊結果。明確 Copy 係唯一剪貼簿效果，而狀態會喺轉語言後保留。route 只因最新視覺擷取受阻先係 `in-progress`。

## Native Password Generator · 原生密碼產生器

`module.passgen` is a live native C++ Password Generator available through `passgen`, `password`, and the canonical route. Its pure C++ core uses Windows BCrypt cryptographic randomness and unbiased rejection sampling for 4–128-character passwords with guaranteed selected classes, optional ambiguity removal and no-repeat enforcement, plus 3–10-word passphrases from the current 252-word dictionary with separator, capitalization, and trailing-digit options. It shows entropy, supports 1–100 rows, retains generated output through language changes, and changes the clipboard only after explicit Copy. The row is `in-progress` solely because visual capture is blocked. · `module.passgen` 係即時原生 C++ 密碼產生器，可以用 `passgen`、`password` 同本體 route 開啟。純 C++ core 用 Windows BCrypt 加密碼學隨機值同無偏 rejection sampling，產生 4–128 字元、保證已揀類別嘅密碼，可選移除易混淆字元同禁止重複；亦可用現有 252 字詞字典產生 3–10 字通行短語，支援分隔符、大寫同尾數字選項。佢會顯示熵值、支援 1–100 行、轉語言時保留已產生輸出，而且只會喺明確 Copy 後先改剪貼簿。清單列只因視覺擷取受阻而係 `in-progress`。

## Native Password Strength · 原生密碼強度

`module.passwordstrength` is a live native C++ local-only Password Strength analyzer available through `passwordstrength`, `pwstrength`, and its canonical route. It calculates ASCII-pool entropy, strength and crack-time bands, and local common-password/repeat/sequence warnings. The editor is masked by default; reveal is explicit and in-memory only, values clear on navigation, and there is no persistence, log, network, or clipboard path. A source guard prevents delayed inactive-editor events from erasing the analysis model. Debug/Release core tests include 11 focused checks; the 224/224 headless UI smoke covers masking, reveal, local warnings, aliases, language retention, accessibility, and horizontal bounds. The row is `in-progress` solely because the inspected 852×880 LowLevel capture had a blank client surface and was discarded. · `module.passwordstrength` 係即時原生 C++、只限本機嘅密碼強度分析器，可以用 `passwordstrength`、`pwstrength` 同本體 route 開啟。佢會計算 ASCII 字元池熵值、強度同破解時間範圍，以及本機常見密碼／重複／序列警告。輸入預設遮蔽；顯示係明確而且只喺記憶體，導覽時會清除，而且冇持久化、記錄、網絡或者剪貼簿路徑。原始碼保護會避免非活動輸入框嘅延遲事件抹走分析模型。Debug／Release core 測試包括 11 個專項檢查；224/224 無頭 UI smoke 覆蓋遮蔽、顯示、本機警告、alias、語言保留、無障礙同水平邊界。呢項只因檢查過嘅 852×880 LowLevel 擷取有空白 client surface、已經丟棄而係 `in-progress`。
