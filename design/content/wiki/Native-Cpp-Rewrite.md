## Current native surface · 目前原生介面

The native shell implements bilingual routing/localization, Dashboard, About, Settings entry, All Apps discovery and live filtering, search, language switching, and all fixed/dynamic route forms. Unported module routes show an honest pending page.

原生 shell 已實作雙語 routing／本地化、Dashboard、About、Settings 入口、所有 app 搜尋同即時篩選、搜尋、轉語言，同全部固定／動態路線格式。未移植模組路線會顯示如實 pending 頁。

Package Manager is now an honest **in-progress** native C++ port, not a pending shell and not parity. It exposes nine views (Discover, Updates, Installed, Bundles, Sources, Ignored, Setup, Settings, and Operations), filters for 11 Windows managers, and read-only result queries for Discover, Updates, and Installed. Discover now locally re-filters cached raw rows by Both name and ID, Package name, Package ID, Exact match, or Show similar packages, with optional case-sensitive and ignore-special-character normalization; it preserves source order and does not start another process, network request, or package query. Details queries render safe parsed common fields above bounded raw output; checked cached Discover/Installed rows can append inert manager/source/ID/version metadata to a bounded Bundle workspace that imports bounded UniGetUI v3-style JSON/`.ubundle` and atomically exports a non-empty workspace, keeps compatible `packages` separate from incompatible audit records, strips non-metadata options, confirms dirty replacement, and never constructs an argv plan or starts a package command; Sources is only a read-only command probe whose raw configuration and diagnostics are withheld until manager-specific secret redaction is proven; Settings persists package-view/search/sort/Discover-filter/update-rule/default-snooze state; and Operations persists preview-only install/update/uninstall plans as structured queue entries with run-next/run-last ordering plus retry-preview markers. Its runtime hardens argument-vector handling, executable/`PATH` resolution, HTTP and captured-output bounds, process-token verification, explicit state, fail-closed integrity checks, and accessibility/UI Automation contracts.

套件管理而家係如實標示為**進行中**嘅原生 C++ 移植，唔再只係 pending shell，亦未達到對等。佢有九個 view（Discover、Updates、Installed、Bundles、Sources、Ignored、Setup、Settings 同 Operations）、11 個 Windows manager filter，同 Discover、Updates、Installed 嘅原生只讀結果查詢。Discover 而家可以喺本機將快取原始資料列用名稱及 ID、套件名稱、套件 ID、完全相同或顯示相似套件重新篩選，亦可選大小寫敏感同忽略特殊字元正規化；會保留來源次序，唔會開另一個 process、網絡要求或者套件查詢。Details 查詢會喺有界原始輸出之上顯示安全已解析常用欄位；已勾選嘅 Discover／Installed 快取資料列可以將惰性管理器／來源／ID／版本 metadata 加入有界 Bundle 工作區，佢會匯入有上限 UniGetUI v3 風格 JSON／`.ubundle` 並原子式匯出非空白工作區、將相容 `packages` 同不相容稽核記錄分開、移除非 metadata 選項、取代未儲存資料前確認，而且絕對唔會建立 argv 計劃或者開套件指令；Sources 只係只讀指令探測，逐管理器機密遮罩未證實之前會刻意隱藏原始設定同診斷；Settings 會保存 package view／搜尋／排序／Discover 篩選／更新規則／預設暫停狀態；Operations 會將只供預覽嘅安裝／更新／解除安裝計劃保存成結構化佇列項目，支援下一個／最後執行排序同重試預覽標記。runtime 已加固 argument vector、executable／`PATH` 解析、HTTP 同擷取輸出上限、process token 驗證、明確狀態、fail-closed integrity 檢查，同 accessibility／UI Automation contract。
