# Native C++ Rewrite · 原生 C++ 重寫

> Foundation milestone, not full parity · 呢個係基礎里程碑，唔係全功能對等

WinForge now has a runnable, self-contained C++20/C++/WinRT WinUI 3 shell beside the shipping managed application. The managed app remains authoritative until every native parity row passes; pending native routes are clearly labelled and never counted as completed features.

WinForge 而家喺發佈中受控 app 旁邊有一個可以執行、自包含嘅 C++20／C++/WinRT WinUI 3 shell。每項原生對等清單通過之前，受控 app 仍然係權威版本；未完成原生路線會清楚標示，絕對唔會當成已完成功能。

## Current native surface · 目前原生介面

The native shell implements bilingual routing/localization, Dashboard, About, Settings entry, All Apps discovery and live filtering, search, language switching, and all fixed/dynamic route forms. Unported module routes show an honest pending page.

原生 shell 已實作雙語 routing／本地化、Dashboard、About、Settings 入口、所有 app 搜尋同即時篩選、搜尋、轉語言，同全部固定／動態路線格式。未移植模組路線會顯示如實 pending 頁。

Package Manager is now an honest **in-progress** native C++ port, not a pending shell and not parity. It exposes nine views (Discover, Updates, Installed, Bundles, Sources, Ignored, Setup, Settings, and Operations), filters for 11 Windows managers, and read-only result queries for Discover, Updates, and Installed. Discover locally re-filters cached raw rows by Both name and ID, Package name, Package ID, Exact match, or Show similar packages, plus optional case-sensitive and ignore-special-character normalization. Cached Discover, Updates, and Installed rows support source-aware, action-scoped multi-select with a delimiter-safe normalized manager/action/ID/source identity. Preview selected install, update, or uninstall adds at most 25 exact argv previews to Operations, clears the transient selection when the query or view changes, and never starts a package command. Checked cached Discover/Installed rows can instead append inert manager/source/ID/version metadata to the bounded Bundle workspace: bounded UniGetUI-v3-style JSON/`.ubundle` import and atomic non-empty export keep compatible packages separate from incompatible audit records, de-duplicate by manager/source/ID/version, strip installation-option/update metadata, confirm dirty replacement, and never construct an argv plan or run a package command. Details queries render safe parsed common fields above bounded raw output; Sources is only a read-only command probe whose raw configuration and diagnostics are withheld until manager-specific secret redaction is proven; and Operations persists preview-only install/update/uninstall plans as structured queue entries with run-next/run-last ordering plus retry-preview markers. Its runtime hardens argument-vector handling, executable/`PATH` resolution, HTTP and captured-output bounds, process-token verification, explicit state, fail-closed integrity checks, and accessibility/UI Automation contracts.

套件管理而家係如實標示為**進行中**嘅原生 C++ 移植，唔再只係 pending shell，亦未達到對等。佢有九個 view（Discover、Updates、Installed、Bundles、Sources、Ignored、Setup、Settings 同 Operations）、11 個 Windows manager filter，同 Discover、Updates、Installed 嘅原生只讀結果查詢。Discover 可以喺本機將快取原始資料列用名稱及 ID、套件名稱、套件 ID、完全相同或顯示相似套件重新篩選，亦可選大小寫敏感同忽略特殊字元正規化。已快取嘅 Discover、Updates 同 Installed 資料列支援識別來源、識別動作嘅多選，並用有長度前綴、已正規化嘅管理器／動作／ID／來源 identity。預覽所選安裝、更新或者解除安裝最多只會加 25 條準確 argv 預覽去 Operations；查詢或者檢視改變時會清除暫時選擇，絕對唔會開套件指令。已勾選嘅 Discover／Installed 快取資料列亦可以將惰性管理器／來源／ID／版本 metadata 加入有界 Bundle 工作區：原子式 UniGetUI v3 風格 JSON／`.ubundle` 匯入／匯出分開相容套件同不相容稽核記錄、按管理器／來源／ID／版本去重、移除安裝選項／更新 metadata、取代未儲存資料前確認，而且絕對唔會建立 argv 計劃或者開套件指令。Details 查詢會喺有界原始輸出之上顯示安全已解析常用欄位；Sources 只係只讀指令探測，逐管理器機密遮罩未證實之前會刻意隱藏原始設定同診斷；Operations 會將只供預覽嘅安裝／更新／解除安裝計劃保存成結構化佇列項目，支援下一個／最後執行排序同重試預覽標記。runtime 已加固 argument vector、executable／`PATH` 解析、HTTP 同擷取輸出上限、process token 驗證、明確狀態、fail-closed integrity 檢查，同 accessibility／UI Automation contract。

The package-manager Settings tab is now a real native persistence slice for package-view, search-text, sort-mode, Discover filter mode/toggles, update-rule state, manager-filter choices, and the default snooze duration. Broader per-manager defaults, backup, and restore stay gated until the remaining settings workflow is ported.

Native Bundles now has an explicit bounded workspace: checked cached Discover and Installed rows append metadata only, while Updates stays preview-only. It atomically exports UniGetUI-v3-style JSON/`.ubundle` (`export_version: 3`), keeps compatible packages separate from incompatible audit records, de-duplicates manager/source/ID/version identity, strips installation-option metadata during import, confirms dirty-workspace replacement, and never creates or runs a package command. · 原生 Bundles 而家有明確有上限嘅工作區：已勾嘅快取 Discover 同已安裝資料列只會附加 metadata，而 Updates 仍然只供預覽。佢會原子式匯出 UniGetUI v3 風格 JSON／`.ubundle`（`export_version: 3`），分開相容套件同不相容審計記錄，按管理器／來源／ID／版本 identity 去重，匯入時移除安裝選項 metadata、取代未儲存工作區前確認，而且絕對唔會建立或者執行套件指令。

套件管理嘅 Settings tab 而家係真實原生持久化片段，會保存套件檢視、搜尋文字、排序模式、Discover 篩選模式／toggle、更新規則狀態、管理器篩選同預設暫停時長。更完整逐管理器預設、備份同還原仍然要等剩低嘅 settings workflow 移植完成。

The native Operations tab now persists structured preview queue entries, imports legacy string history, lets the user clear them, and exposes run-next/run-last/retry-preview controls. The real mutation coordinator, output capture, cancellation, and completed/live mutation parity remain gated.

原生 Operations tab 而家會保存結構化預覽佇列項目、匯入舊字串歷史、可以清除，並提供下一個／最後執行／重試預覽控制。真正修改協調器、輸出擷取、取消同完成／即時修改對等仍然鎖住。

Check Digit Validator is the next real native utility slice. `checkdigit`, `luhn`, and `module.checkdigit` now open a live standard-C++ implementation of Luhn/card-brand detection, ISBN-10/13, EAN-13, UPC-A, and incremental mod-97 IBAN validation. It preserves input, scheme, and results across language changes, uses no CLR or external side effects, and exposes localized names plus a polite live-region accessibility contract.
