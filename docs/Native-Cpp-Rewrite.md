### Native safe regex search and builder · 原生安全正規搜尋同建立器

The native C++ shell uses PCRE2-16 for Shell catalog search, All Apps, and Package Manager cached Discover filtering. Every pattern has UTF/UCP semantics and strict interactive limits (512 UTF-16 pattern units, 8 KiB input, nesting/code-size/match/depth/heap limits, and a 10 ms callout deadline); JIT is not used. Shell and All Apps test each metadata field independently, so anchors retain field scope; invalid All Apps patterns retain the previous valid list.

原生 C++ shell 會用 PCRE2-16 處理 Shell 目錄搜尋、所有 app 同 Package Manager 已快取 Discover 篩選。每個模式都有 UTF/UCP 語義同嚴格互動限制（512 個 UTF-16 模式 units、8 KiB 輸入、nesting／code-size／match／depth／heap 限制同 10 ms callout deadline）；唔用 JIT。Shell 同所有 app 會分開比對 metadata 欄位，所以 anchor 保留欄位範圍；所有 app 無效模式會保留之前有效清單。

`module.regextester` is now a real full four-step native builder: target/flags; escaped literal recipes (contains, exact, prefix, suffix, whole word) plus route/package/semantic-version starters and safe literal/class/common-token composition; group/alternation/word-boundary/lookaround assertion/quantifier composition; then match/capture preview and Apply. The registry pins its three targetable native search surfaces; Apply stays disabled for an invalid pattern. It can apply to Shell, All Apps, cached Package Discover, or remain tester-only. It is not a claim of complete .NET regex or replacement parity.

`module.regextester` 而家係真正完整四步原生建立器：目標／旗標；已 escape literal recipe（包含、完全、開頭、結尾、完整字）連路線／套件／語義版本起始式，同安全 literal／class／常用 token 組合；group／alternation／word-boundary／lookaround assertion／quantifier 組合；之後 match／capture 預覽同 Apply。registry 釘實三個可以套用嘅原生搜尋位置；模式無效時 Apply 會保持停用。佢可以套用去 Shell、所有 app、已快取 Package Discover，或者只留喺測試器。呢個唔係完整 .NET regex 或 replacement 對等聲稱。

Package Discover regex is local cached-result filtering only: it is never copied into argv or sent to HTTPS, cannot start a package query while active, disables external Search, and exposes a monotonic `NativePackageQueryAudit` epoch that changes only when a genuine CLI/HTTPS query starts. A builder application validates before navigation, forces Discover instead of retaining an Updates/Installed view, and retains rows only from a completed Discover search cache. ## Safety and compatibility gates · 安全同相容閘門

The native port must preserve DPAPI secret compatibility, atomic persistence, normal-integrity-only update/install paths, fail-closed elevation gates, protocol compatibility, and every reactor invariant: cold-shutdown boot, default-off real shutdown, abortability, reversible real-world effects, and waste disk/cap limits.

原生移植一定要保留 DPAPI secret 相容、原子式 persistence、只限正常 integrity 嘅更新／安裝路徑、fail-closed 提權 gate、protocol 相容，同埋所有反應堆 invariant：cold-shutdown 開機、真實關機預設關閉、可以取消、真實副作用可還原、廢料磁碟／容量上限。

## Cutover definition · 正式切換定義

The rewrite is complete only when every route, feature, control, service, companion, external launcher, updater, installer, SDK protocol, test project, document, wiki page, GitHub Pages mirror, and changed screenshot has passing evidence. Final binary inspection must show no `coreclr`, `hostfxr`, managed assembly, C++/CLI metadata, managed subprocess, or wrapper dependency. Only then may the managed application be retired.

只有每條路線、每項功能、每個 control、service、companion、外部 launcher、updater、installer、SDK protocol、測試專案、文件、wiki、GitHub Pages 鏡像同改過頁面嘅截圖全部有通過證據，先叫完成。最後 binary inspection 必須證明冇 `coreclr`、`hostfxr`、受控 assembly、C++/CLI metadata、受控 subprocess 或 wrapper 相依；到嗰陣先可以退役受控 app。
